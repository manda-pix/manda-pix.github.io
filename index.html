Vulnerability Management Project

Using VMware workstation and Nessus Essentials, I ran a Windows 10 machine in VMware and installed a deprecated version of Firefox. Then using Nessus Essentials, I scanned the system for vulnerabilities and to see what it would show with the deprecated software. Nessus shows the vulnerabilities and gives details on how to fix them. To mitigate some of the vulnerabilities, I updated the software on the Windows system then ran Nessus again and it showed less vulnerabilities.

Testing the Security of Accounts

Using a Kali Linux virtual machine in VirtualBox, I created 5 users account with differing password complexities. Then ran the John the Ripper attack tool to see which passwords were compromised.

Configuring Active Directory Project

Using VirtualBox, I set up an Active Directory lab. The lab consists of 2 Windows 10 machines and 1 Windows Server 2019 machine. The Server machine housed the domain controller which had 2 NICs, one for the internal network and the other for the external network. This project was a way to get a better and more hands on experience in configuring active directory.

Conducting Incidence Response

This was a lab simulating captured evidence of a security incident and I went through and analyzed the contents of a PCAP file using NetWitness Investigator to find any stolen data. Then using the Electronic Evidence Examiner, I imported the disk image of a suspect that was provided to find any evidence that would connect them to the security incident.

Configuring GRUB Security

Using CentOS on a VirtualBox machine, the goal was to secure the boot process GRUB 2. To do so, I hide the GRUB menu by configuring the default file and then in the user script file /etc/grub.d/01_users , I created a super user and a regular user that had their passwords encrypted in the file.
